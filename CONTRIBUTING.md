# Contributing to the Project

First off, thank you for checking out the project! Whether you are fixing a typo, patching a minor bug, or proposing a massive new feature, community involvement is what keeps independent projects alive and thriving.

We want to build a sustainable, open ecosystem, and we welcome your help to get there.

---

## License Choice & Open Source Commitment

This project is fully open-source and proudly distributed under the **GNU Affero General Public License v3.0 (AGPL-3.0)**.

We chose the AGPL-3.0 specifically because we believe in true open-source sustainability. It ensures that the software remains free, transparent, and modifiable for everyone who uses it, while guaranteeing that any improvements made by the community are shared back openly with the ecosystem.

---

## Developer Certificate of Origin (DCO)

To keep the project legally clean and ensure a safe environment for everyone using the software, we use a standard DCO workflow. By submitting a Pull Request, you certify that:

1. **Original Work:** The contribution is entirely your own original creation, or you have the clear legal right and authorization to submit it under the project's active licensing framework.
2. **Project Stewardship:** You grant **Speetjens Media** a perpetual, worldwide, non-exclusive, no-charge, royalty-free, irrevocable copyright license to reproduce, prepare derivative works of, publicly display, and distribute your contributions.

## Licensing & Cloud Notice

By contributing to this repository, you agree that your contributions will be licensed under the project's AGPL-3.0 framework to the public.

We believe in leading by example: **Speetjens Media** operates under these exact same open-source principles. We pledge that every single feature, update, and enhancement developed for the software will be committed directly to this public open-source repository. No features will ever be gatekeepered or locked behind a paywall; the self-hosted version will always remain completely free and fully featured.

The sole exception to this is the specialized orchestration and infrastructure code required to run the platform as a managed cloud service. This specific structure allows us to protect the ecosystem from other companies profiting off our collective labor, and ensures we retain the exclusive right to provide managed cloud distributions down the road. Our goal for the cloud version is to keep access as close to free as possible for everyday users-potentially offsetting operational costs through non-intrusive, privacy-respecting means—while ensuring the platform's infrastructure and long-term maintenance are fully sustained.

Self-hosting will always be free and maintained regardless of the cloud platform with open source code being available to the public and all code being community driven and made public regardless of circumstance.

## How to Get Involved

### 1. Report Bugs & Request Features

If you find a bug or have an idea for an improvement, please open a GitHub Issue! Before opening a new issue, please do a quick search to see if someone else has already reported it.

### 2. Submitting a Pull Request (PR)

When you're ready to contribute code, follow these quick guidelines:

- **Fork & Branch:** Fork the repository and create your branch from `main`.
- **Keep it Focused:** Try to keep your PRs tightly scoped to a single fix or feature so it’s easy to review.
- **Code Style:** Ensure your code matches the existing style and conventions of the codebase.

Thank you again for your support, your time, and for helping us protect independent open-source development!
